# AI 视频生成指南（Seedance 2.0 专用）

## 一、生成流水线总览

```
角色参考图（已确认） → 分镜关键帧图 → 图生视频（I2V） → 剪辑合成
```

**前置条件**：开始视频生成前，确保以下素材已就绪：
1. 每个角色至少有一张已确认的正面参考图
2. 每个分镜至少有一张已确认的关键帧图
3. 分镜脚本中已标注时长、运镜、画面描述

---

## 二、Seedance 2.0 核心能力

| 能力 | 参数 |
|------|------|
| 输出分辨率 | 原生 720p，上采样至 1080p/2K |
| 单段最长时长 | 15 秒 |
| 参考文件上限 | 12 个（最多 9 张图 + 3 个视频 + 3 个音频） |
| 支持宽高比 | 16:9、9:16、1:1、21:9、4:3、3:4 |
| 角色一致性 | 正面→正面 95%、正面→侧面 85%、换场景 80% |
| 音视频同步 | 原生对口型，支持 8+ 语言 |
| 多镜头叙事 | 单次生成内支持自动分镜和镜头切换 |
| 物理感知 | 理解重力、摩擦力、碰撞、材质属性 |

## 三、提示词万能公式

所有视频提示词遵循以下结构：

```
主体 + 动作 + 场景 + 光影 + 运镜 + 风格 + 画质 + 约束
```

### 结构化模板格式

```
主体：[人物/物体，年龄、服装、外貌特征]
动作：[连续、自然的动作描述]
场景：[环境、时间、天气、氛围]
运镜：[景别] + [运动方式] + [角度/焦距]
风格：[视觉锚点：导演风格/电影风格], [光线], [色彩]
约束：[稳定性约束], [画质标签], [时长]
```

### 各段详解

#### 1. 主体（Subject）——开头最重要

**权重最高的位置**，必须在提示词开头立即锁定主体。

**中文写法**：
```
@角色1（林墨），32岁男性，炭灰色西装，棱角分明的下颌，后梳黑色短发
```

**英文写法**：
```
@Image1 (Lin Mo), a 32-year-old man in charcoal suit, sharp jawline, slicked-back hair
```

**要点**：
- 使用 @标签 引用角色参考图
- 即使有参考图，仍需文字描述核心特征作为冗余锚定
- 角色限制在 1-2 人，超过 2 人容易身份混淆

#### 2. 动作（Action）——写慢、写连续、写具体

Seedance 2.0 对动作的细腻程度理解极高，但需要正确的写法。

**核心原则**：
- **一个分镜一个主要动作**：多个运动动词会让模型混乱
- **动作写慢、写连续**：慢动作最容易出精品
- **标注物理细节**：材质、重力、风力效果

**好的中文写法**：
```
缓缓转身，目光从窗外移向门口，风衣下摆随动作轻轻摆动
```

```
轻轻抬起右手整理领带，然后转头望向窗外，动作自然流畅
```

**差的写法**：
```
走来走去（太模糊）
快速奔跑并跳起（太剧烈）
```

**推荐动作修饰词**：缓慢、轻柔、连贯、自然、流畅、平稳、微微、不僵硬

**避免的描述**：夸张、高速、剧烈、复杂多人互动、大幅度扭曲

**物理感知描述提升真实感**：
```
丝绸裙摆在微风中轻轻飘动，较重的裙边自然下垂
玻璃杯从桌沿滑落，碎片以真实的动量向外散射
```

#### 3. 运镜（Camera）——写稳、写简单

**核心原则**：复杂运镜交给后期剪辑，生成时求稳。

| 运镜 | 中文提示词 | 英文提示词 | 适用场景 |
|------|-----------|-----------|---------|
| 推镜 | 缓慢推镜，从中景推至特写 | `slow dolly-in from medium to close-up` | 情感揭示、紧张 |
| 拉镜 | 镜头缓缓拉远展现全景 | `slow dolly-out to reveal environment` | 全景、结局 |
| 横摇 | 镜头平稳从左向右摇动 | `smooth pan from left to right` | 环境展示 |
| 环绕 | 环绕主体180度 | `orbit 180 degrees around subject` | 角色出场 |
| 跟拍 | 稳定器平稳跟拍 | `gimbal tracking alongside subject` | 跟随、行走 |
| 升降 | 镜头从地面升至俯瞰 | `crane up from ground to aerial` | 格局、命运 |
| 手持 | 手持镜头，轻微自然晃动 | `handheld, natural slight shake` | 紧张、追逐 |
| 固定 | 固定镜头，画面稳定 | `static camera, locked off` | 对话、沉淀 |

**运镜速度搭配**：用 `缓慢` / `中速` / `快速` 配合，如 `缓慢推镜`

**复合运镜**：按时间拆分描述
```
开头缓慢推镜3秒，然后固定镜头保持5秒
```

#### 4. 风格（Style）——锁定视觉调性

选对风格词可一句话锁定画面调性，防止风格漂移。

| 题材 | 中文风格词 | 英文风格词 |
|------|-----------|-----------|
| 日常/甜宠 | 治愈清新、唯美文艺、暖色调 | `healing fresh, soft warm tones, dreamy` |
| 霸道总裁 | 电影质感、暗调高级、冷色调 | `cinematic noir, dark premium, cool tones` |
| 悬疑 | 黑色电影、高对比度、冷色调 | `film noir, chiaroscuro, high contrast` |
| 古风 | 中国古典美学、华丽质感、烛光 | `traditional Chinese aesthetic, rich textures` |
| 科幻 | 赛博朋克、霓虹灯光、未来感 | `cyberpunk, neon lighting, futuristic` |

**导演风格锚点**：引用知名导演/电影风格精确定调（英文提示词更有效）
- 王家卫风格 → 霓虹色、慢动作、孤独感
- 韦斯·安德森风格 → 对称构图、糖果色
- 大卫·芬奇风格 → 暗调、精确、不安

#### 5. 约束（Constraints）——必加，保险丝

**Seedance 2.0 不支持负面提示词（Negative Prompt）**，必须全部使用正向约束。

**中文必加约束词（焊死在提示词末尾）**：
```
面部稳定不变形，人体结构正常，比例自然，
动作自然流畅不僵硬，同一角色服装一致发型不变，
画面稳定不闪烁不抖动，4K高清，细节丰富，色彩自然。
```

**英文约束词**：
```
Maintain face and clothing consistency, no distortion,
natural human anatomy, smooth natural movements,
stable picture, no flickering, 4K Ultra HD, rich details, sharp clarity.
```

**根据场景增加的约束**：
- 不想要字幕：`生成视频不带字幕`
- 防背景闪烁：`背景稳定不闪烁` / `no background flickering`
- 防画面抖动：`画面稳定如三脚架` / `tripod-stable`

## 四、参考系统（@标签语法）

Seedance 2.0 最强大的功能之一是多模态素材引用。上传素材后通过 @标签 精确指定用途。

### 图像参考

```
@图片1 — 角色 A 正面参考图（锁定身份）
@图片2 — 角色 B 正面参考图
@图片3 — 场景参考图
@图片4 — 关键帧构图参考
```

### 视频参考

```
@视频1 — 运镜参考（模型将复制此视频的镜头运动）
@视频2 — 动作参考（模型将模仿此视频的人物动作）
```

### 音频参考

```
@音频1 — 背景音乐（模型将节奏同步到音频节拍）
@音频2 — 对白音频（模型将进行对口型）
```

### 常用 @引用写法

| 用途 | 中文写法 | 英文写法 |
|------|---------|---------|
| 指定首帧 | `@图片1 作为首帧` | `@Image1 as starting frame` |
| 指定角色 | `@图片1 的女生作为主角` | `@Image1 as the protagonist` |
| 参考运镜 | `参考@视频1的运镜和转场` | `Reference @Video1 for camera movement` |
| 参考动作 | `模仿@视频1的动作` | `Mimic the actions from @Video1` |
| 指定配乐 | `@音频1 用于配乐` | `@Audio1 for background music` |
| 视频延长 | `将@视频1延长15秒` | `Extend @Video1 for 15 seconds` |
| 元素替换 | `将@视频1中的男主换成@图片2` | `Replace male lead in @Video1 with @Image2` |

### 参考文件选择原则

| 用途 | 选择标准 |
|------|---------|
| 角色锁定 | 清晰正面照，单人，占画面 60-80%，无遮挡，至少 1080p |
| 场景锁定 | 宽幅全景，展示完整空间，光影氛围到位 |
| 运动参考 | 截取关键片段，越短越精准（3-5 秒最佳） |
| 构图参考 | 展示目标构图的图片（可以是电影截图） |
| 音频参考 | 清晰无压缩伪影，节奏点明确 |

## 五、分镜视频提示词格式

### 单动作 I2V 提示词（中文，推荐）

```
【分镜 01】时长：5s | 宽高比：16:9

@图片1（林墨参考图），32岁男性，炭灰色西装，
站在办公室落地窗前，望着窗外城市天际线，
缓缓转身面对镜头，表情从沉思变为冷峻坚定，
缓慢推镜，从中景推至面部特写，
窗户透入冷色调光线，在人物边缘形成轮廓光，
电影质感，去饱和冷色调配暖色城市灯光背景，
面部稳定不变形，动作自然流畅，画面稳定，4K高清。
```

### 叙事段落 I2V 提示词（中文）

```
【分镜 05】时长：12s | 宽高比：16:9

@图片1（林墨参考图），@图片2（苏婉参考图）

第一个画面（0-4s）：
@图片1 林墨站在办公室落地窗前，背对镜头，
缓缓转身180度面向门口方向，
城市夜景从窗外透入冷蓝色光线。

切换到 —

第二个画面（4-8s）：
双人中景，@图片1 和 @图片2 隔着办公桌对视，
@图片1 表情冷峻，@图片2 神情坚定微微抬起下巴，
固定镜头，85mm人像镜头效果，侧光打在两人之间。

切换到 —

第三个画面（8-12s）：
@图片2 苏婉面部特写，
眼睛微微眯起带着决意，嘴唇微张似要开口，
缓慢推镜，浅景深，背景逐渐虚化，
面光照亮面部，与暗色背景形成对比。

电影质感，面部稳定不变形，人体结构正常，
动作自然流畅不僵硬，画面稳定不闪烁，4K高清。
```

### 全模态参考提示词（图 + 视频 + 音频）

```
参考@视频1的运镜手法和转场节奏，
生成@图片1中穿黑衣的角色在竹林中拔剑出鞘的视频。
起始帧的视角和景别严格参照@视频1，
拔剑出鞘后慢动作对焦剑身，背景人物虚化。
仅生成打斗音效和环境音效，
并加入背景音乐@音频1。
面部清晰不变形，动作自然流畅，画面稳定，4K高清。
```

## 六、图生视频（I2V）专项技巧

### 关键帧选择标准

| 标准 | 要求 |
|------|------|
| 分辨率 | ≥ 1280×720（推荐 1920×1080+） |
| 主体占比 | 30-70% 画面面积 |
| 动态潜力 | 物体处于即将运动的状态（如手臂微抬、嘴微张） |
| 光影自然 | 色彩饱和度适中，纹理细节清晰 |
| 角度合理 | 避免极端俯仰角（不超过 45°） |
| 空间清晰 | 前后景关系明确 |

### I2V 提示词要点

1. **描述从图片当前状态出发的运动**：告诉模型接下来发生什么
2. **锁定静态元素**：`背景保持不动` / `background remains static`
3. **补充图片未能体现的信息**：氛围、物理效果、声音暗示
4. **保持风格一致**：使用与图片生成时相同的风格关键词

### I2V 图生视频提示词模板

**中文模板**：
```
基于参考图保持人物样貌与服装一致，
[具体动作描述，写慢写连续]，
[运镜描述]，[光影氛围]，
动作自然流畅不僵硬不变形，
稳定运镜，高清细节，电影质感，
面部稳定不变形，画面稳定，4K高清。
```

## 七、时长策略

### 先短后长原则

**先生成 5 秒确认效果，满意后再延长到 10-15 秒。**

| 时长 | 适用场景 | 提示词策略 |
|------|---------|-----------|
| 4-7s | 单一动作、反应镜头、空镜头 | 专注一个明确的动作或瞬间 |
| 10-15s | 多镜头叙事、对话、复杂段落 | 明确起-承-转结构，用转换词分隔 |

### 超过 15 秒的内容

将最后一帧作为下一段生成的参考图，创建无缝延伸：

```
第 1 段（0-15s）→ 提取最后一帧 → 作为第 2 段的 @图片1 → 第 2 段（15-30s）
```

## 八、转场效果

| 转场 | 中文写法 | 英文写法 | 适用场景 |
|------|---------|---------|---------|
| 硬切 | 切换到 | Cut to | 对话正反打、紧张节奏 |
| 淡入淡出 | 黑屏过渡后淡入 | Fade to black, then fade in | 时间跳跃 |
| 溶解 | 溶解过渡到 | Dissolve to | 回忆、梦境 |
| 匹配剪辑 | 从[A]匹配切换到[B] | Match cut from [A] to [B] | 动作连续 |

**原则**：同一短剧内转场风格保持统一。日常叙事多用硬切，情感段落可用溶解/淡出。

## 九、视频生成自检清单

- [ ] 每个分镜是否只有一个主要运镜动作？
- [ ] 是否包含场景标签 `【场景：{场景名}·{变体名}】`？
- [ ] 角色参考图是否已在 `references` 字段中 @标记？
- [ ] 光影描述是否与场景变体的时间/天气一致？（黄昏用暖光，夜晚用月光/烛光，日景用阳光）
- [ ] 有台词的分镜是否包含对话内容？（音效后缀应为「保留环境声和对白」）
- [ ] 动作描述是否使用了缓慢/自然/流畅等修饰词？
- [ ] 提示词是否至少包含一个连续的运动描述？
- [ ] 时长和宽高比是否标注？
- [ ] 是否附加了必加约束词（面部不变形、画面稳定等）？
- [ ] 提示词长度是否在 30-200 词范围内？
- [ ] 角色是否控制在 2 人以内？
- [ ] 静态元素是否有「保持不动」标注？
- [ ] 关键帧图分辨率是否达标（≥ 1280×720）？
- [ ] 叙事段落是否使用了明确的转换词（切换到 / Cut to）？

## 十一、提示词迭代与排错

### 迭代工作流

```
初始提示词 → 生成 2-3 个变体 → 评估 → 调整提示词 → 再次生成 → 确认
```

**核心原则**：每次迭代只修改 1 个变量，以便定位哪个修改生效。

### 迭代排错决策树

| 问题 | 解决方法 |
|------|---------|
| 构图不对但动作正确 | 只调运镜（景别 + 运动方式），保持主体和动作不变 |
| 运动太晃/太快 | 互换 `手持` ↔ `稳定器`，指定速度，不动其他 |
| 风格/色彩漂移 | 用更强的视觉风格锚点替换，减少形容词 |
| 手部/标签/光晕重复出现 | 换约束词或换景别，近景换中景试试 |
| 多角度混乱 | 把「多角度」改为「单一跟拍镜头」 |
| 角色面部变形 | 更换更清晰的正面参考图，加强约束词 |
| 运动幅度不足 | 增强动作动词：「缓慢」→「果断」 |
| 运动幅度过大 | 添加约束：「轻柔」、「微微」、「克制」 |
| 背景意外变化 | 添加「背景保持完全不动」 |
| 角色数量错误 | 简化为最多 2 个角色，明确 @标记 |
| 运镜不生效 | 将运镜描述提到提示词前部 |
| 画面风格不统一 | 附加项目风格圣经后缀 |

### 质量评估标准

每段生成的视频按以下维度 1-5 分评估：

| 维度 | 1 分（不可用） | 3 分（可接受） | 5 分（优秀） |
|------|-------------|-------------|-------------|
| 角色一致性 | 面部完全不同 | 大致相似 | 与参考图高度一致 |
| 运动自然度 | 抽搐、变形、穿模 | 基本自然 | 流畅自然如实拍 |
| 构图准确度 | 与描述不符 | 大致正确 | 精确匹配分镜 |
| 光影一致性 | 与项目风格不符 | 基本匹配 | 完美匹配风格圣经 |
| 物理合理性 | 严重穿模 | 偶有小问题 | 符合物理规律 |

**通过阈值**：所有维度 ≥ 3 分即可使用，关键镜头（特写、高潮）应达到 4+ 分。

## 十二、避雷指南

### 千万别写

| 类型 | 说明 |
|------|------|
| 复杂多人互动 | 打架、握手、拥抱等容易穿模 |
| 模糊主观词 | 「好看」「很美」「很酷」等于没描述 |
| 矛盾要求 | 「超高速奔跑」+「极度稳定无模糊」 |
| 负面提示词 | 不要写 `no ugly`、`not blurry` 等否定句式 |
| 敏感内容 | 色情、暴力、危险动作、版权人物 |

### 提示词写法对照表

| 想避免的问题 | 错误写法 | 正确写法（正向约束） |
|-------------|---------|-------------------|
| 模糊 | 不要模糊 | 画面清晰锐利，细节丰富 |
| 变形 | 不要变形 | 面部稳定不变形，人体结构正常 |
| 多余肢体 | 不要多手 | 人体比例自然，正常手臂和腿 |
| 水印/文字 | 不要水印 | 画面干净，不带字幕和文字 |
| AI 塑料感 | 不要假 | 皮肤纹理自然，有毛孔细节 |
| 平光 | 不要平光 | 有方向感的戏剧性光线，明暗分明 |
| 画面抖动 | 不要抖 | 画面稳定如三脚架，丝滑流畅 |

## 十三、后期合成指导

### 合成流水线

```
AI 生成片段 → 粗剪（顺序拼接） → 精剪（节奏调整） → 配音/字幕 → 音效/BGM → 调色 → 输出
```

### 各阶段要点

**1. 粗剪**
- 按分镜脚本顺序拼接所有视频片段
- 检查镜头间过渡是否自然
- 标记需要重新生成的问题片段

**2. 精剪**
- 调整每个镜头的入点和出点
- 删除首尾帧的不自然运动（AI 视频首尾帧常有瑕疵）
- 控制节奏：高潮段落加快切换，情感段落延长停留

**3. 配音与字幕**
- AI 配音：使用 TTS 工具生成角色对白
- Seedance 2.0 的 @音频 功能可在生成时直接传入对白音频实现对口型
- 字幕：自动生成后人工校对

**4. 音效与 BGM**
- BGM 与情绪曲线匹配
- 环境音效增强真实感（脚步声、开门声、雨声）
- Seedance 2.0 可原生生成音效，也可后期添加

**5. 调色**
- AI 生成片段间可能有轻微色差，需统一调色
- 使用 LUT 或调色预设保持全片一致
- 确保与项目风格圣经的色调一致

### 推荐工具

| 阶段 | 工具 |
|------|------|
| 视频剪辑 | 剪映/CapCut、Premiere Pro、DaVinci Resolve |
| AI 配音 | 11ElevenLabs、通义听悟、剪映 TTS |
| 字幕生成 | 剪映自动字幕、Whisper |
| 音效库 | Epidemic Sound、Freesound |
| 调色 | DaVinci Resolve、Premiere Pro Lumetri |

## 十四、视频提示词质量优化 10 维度

生成视频提示词后，按以下 10 个维度逐一检查和优化。

### 1. 台词嵌入动作描述

将对白直接嵌入角色动作中，而非仅放在独立的 dialogue 字段。

```
// 错误
顾长安手持酒杯缓缓啜饮，用看似随意的语气开口说话
// 正确
顾长安端起酒杯从容啜饮，放下酒杯，用看似随意的语气说:"你的断剑上这个符号——我在别的地方也见过。"
```

**规则**：每个分镜中有台词的角色，必须在视频提示词中嵌入至少核心台词。格式为 `角色+动作+说:"台词"`。

### 2. 精确时间戳分段

为多镜头分镜添加时间戳，为单镜头标注景别和时长。

```
// 错误
第一个画面：... 切换到 第二个画面：...
// 正确（多镜头）
0-5秒(中景，正面平视): ... 5-10秒(全景升镜，仰角): ... 10-15秒(跟随镜头推至特写): ...
// 正确（单镜头）
分镜1(近景3秒)侧拍平视。
```

### 3. 音效描述层

每个分镜必须包含环境音或动作音效描述。

**常用音效词库**：

| 场景类型 | 音效词 |
|---------|--------|
| 武打 | 剑鸣、刀风、拳击肉声、骨折声、气劲音爆 |
| 环境 | 风声、雨声、溪流声、蝉鸣、夜虫 |
| 室内 | 脚步声、烛火声、门扉吱呀、酒杯碰击 |
| 情绪 | 心跳声（紧张时）、寂静（反差前） |

### 4. 每段独立机位标注

多镜头分镜的每个子段必须独立标注景别和机位。

```
// 错误
多镜头叙事，运镜：固定镜头+缓慢推镜
// 正确
0-5秒(中景5秒)正面平视。 5-8秒(近景3秒)对切平视。 8-12秒(特写4秒)侧拍微俯+缓慢推镜。
```

### 5. VFX 描述（动作场景）

武打和特效场景必须包含视觉效果描述。

**常用 VFX 词库**：

| 类型 | 效果词 |
|------|--------|
| 剑气/刀气 | 发光轨迹残影、气流扰动、冲击波 |
| 破坏 | 碎石纷飞、木屑飞溅、尘土弥漫 |
| 光效 | 体积光、丁达尔效应、光柱、光斑 |
| 渲染 | 虚幻引擎5渲染、光线追踪、全局光照 |

### 6. 速度词多样化

避免"缓缓"过度使用，根据场景情绪匹配速度词。

| 速度 | 可用词 |
|------|--------|
| 极快 | 瞬间、骤然、猛地、电光石火间、一闪而过 |
| 快速 | 果断、利落、迅速、敏捷地 |
| 适中 | 稳步、从容、不疾不徐 |
| 缓慢 | 缓缓、徐徐、慢慢 |
| 极慢 | 凝滞般、如同静止、一帧一帧地 |
| 变速 | 由慢至快、骤然加速、戛然而止 |

**规则**：同一分镜内"缓缓"最多使用 2 次。武打场景必须有快慢对比。

### 7. 情绪微表情细化

用具体的面部微动作替代概括性情绪描述。

| 情绪 | 微表情描述 |
|------|-----------|
| 震惊 | 瞳孔骤缩、嘴唇微颤、身体微僵 |
| 愤怒 | 眉峰倒竖、鼻翼微张、指节发白 |
| 悲伤 | 眼眶微红、睫毛轻颤、喉结上下滚动 |
| 喜悦 | 眼角细纹舒展、嘴角不自觉上扬、瞳孔微亮 |
| 警觉 | 目光一凝、肩膀微沉、手指无意识摸向剑柄 |
| 犹豫 | 目光闪烁、嘴唇微启又合上、手指无意识摩挲 |

### 8. 创作自由度指令

在提示词末尾（质量约束词之前）加入一句创作自由度指令。

```
允许优化镜头运动和构图，整体画面充满武侠电影质感，唯美而充满力量感。
合理安排镜头包含近景中景全景和特写，自由发挥动态构图。
```

### 9. 动态光影变化

描述光影在镜头内的变化过程，而非仅描述静态光线。

```
// 错误
烛光暖色调
// 正确
烛火跳动在墙壁投下摇晃的影子，每一次跳动都让两人面容上的明暗交替变幻

// 错误
月光银色
// 正确
月光从云层缝隙间流泻而下，随云层飘动在地面形成移动的光斑
```

### 10. 服饰发丝道具的动态描述

将静态服饰描述转为动态运动描述。

| 元素 | 动态描述示例 |
|------|------------|
| 衣袂/袍角 | 随风飘动、被剑气卷起、贴身飞扬 |
| 发丝 | 随风飘散、被气浪吹起、沾着血滴贴在脸上 |
| 武器 | 剑身反射光芒、刀刃切割空气、剑穗飞扬 |
| 饰品 | 玉佩轻晃、铃铛清响、面纱飘动 |

## 十五、视频时长与分镜衔接规范

### 1. 硬性时长限制

**即梦(Seedance)视频最长 15 秒**。所有分镜的 `duration_seconds` 不得超过 15 秒。

### 2. 段落时间精确匹配

每个分镜内所有段落的时间之和必须**精确等于** `duration_seconds`。

```
// 正确 (duration_seconds = 15)
分镜1(中景9秒) ... 分镜2(特写4秒) ... 分镜3(特写2秒) ... → 9+4+2=15 ✅
0-7秒(...) 7-11秒(...) 11-15秒(...) → 15 ✅

// 错误
分镜1(中景5秒) ... 分镜2(特写5秒) ... 分镜3(特写5秒) ... → 15 但 duration=12 ❌
```

### 3. 内容加权时间分配

段落时长不应平均分配，应按**内容权重**分配：

| 内容类型 | 权重系数 | 说明 |
|---------|---------|------|
| 台词 | 1.2x 字数 | 说话需要展示时间，每个汉字约 0.3 秒 |
| 场景描述 | 0.3x 字数 | 视觉内容展示较快 |
| 动作 | 0.5x 字数 | 动作需要适当展示时间 |

**规则**：
- 每段最少 2 秒
- 台词密集段落分配更多时间
- 纯视觉/反应段落可压缩至 2-3 秒

### 4. 台词容量限制与裁剪

当台词总阅读时间超过可用预算（duration × 50%）时，必须精选核心台词：

**精选优先级**：
1. 首句台词（建立对话）
2. 末句台词（结论/转折）
3. 含关键剧情信息的台词（角色名、事件、真相等）
4. 带强情绪标记的台词（？！、动作方向）

**阅读时间估算**：汉字数 ÷ 3.5 + 台词条数 × 0.5 秒

### 5. 相邻分镜衔接

同一集内相邻分镜，根据前一分镜的 `transition_type` 添加衔接标记：

| transition_type | 标记 | 使用场景 |
|----------------|------|---------|
| `fade_black` | `【渐黑转场】` | 场景大切换（如室外→室内） |
| `dissolve` | `【叠化转场】` | 柔和过渡（如时间流逝） |
| `fade_white` | `【渐白转场】` | 高光/爆发场景衔接 |
| `cut` | 无标记 | 同场景内直接剪切 |

**标记位置**：放在下一分镜第一段内容的最开头。

### 6. 时长与分镜数量参考

| duration_seconds | 建议分镜数 | 适合场景 |
|-----------------|-----------|---------|
| 5-8 秒 | 1 个单镜头 | 情绪特写、环境建立 |
| 8-10 秒 | 1-2 段 | 简短对话、单一动作 |
| 10-12 秒 | 2-3 段 | 标准对话、动作序列 |
| 12-15 秒 | 2-4 段 | 复杂对话、史诗动作 |

### 7. 场景标签规范

每个分镜视频提示词的 `prompt` 字段中**必须包含场景标签**，格式为 `【场景：{场景名}·{变体名}】`。

**位置规则**：场景标签放在提示词第一行（风格头）之后、分镜内容之前。

```
// 正确格式
智能分镜，武侠古装剧，动作流畅自然。
【场景：天机门·大殿内】
分镜1(中景→特写6秒)...

// 错误：缺少场景标签
智能分镜，武侠古装剧，动作流畅自然。
分镜1(中景→特写6秒)...
```

**场景标签来源**：从分镜关联的 `scene_variant` 获取，格式为 `场景名·变体名`。

**作用**：
- 为视频生成模型提供明确的空间环境语境
- 确保生成画面与场景设定一致（光线、氛围、建筑等）
- 便于后期批量检索同场景分镜

### 时长匹配速查

| 检查项 | 通过标准 |
|--------|---------|
| duration ≤ 15s | 不超过即梦上限 |
| 段落总和 = duration | 精确匹配 |
| 每段 ≥ 2s | 无过短段落 |
| 台词时间 ≤ 50% duration | 台词不超载 |
| 场景切换有转场标记 | 衔接流畅 |
| 含场景标签 | 每条提示词含 `【场景：...】` |

## 十六、提示词展示与复制规范

### 系统展示要求

在管理平台的分镜详情页中，视频提示词的展示必须包含完整的可用信息，使用户复制后可直接粘贴到即梦使用。

### 复制内容构成

| 组成部分 | 来源 | 必须包含 |
|---------|------|---------|

| 角色参考图引用 | `video_prompt.references` | ✅ 是 |

| 宽高比 | `video_prompt.aspect_ratio` | ✅ 是 |
| 台词内容 | 嵌入在 prompt 中 | ✅ 是（需验证） |
| 场景标签 | 嵌入在 prompt 中 | ✅ 是（需验证） |

### 提示词完整性验证

复制功能应在输出前验证以下内容确实存在于 prompt 文本中：

```
1. 场景标签：检查 prompt 是否含 `【场景：` 开头的标签
2. 台词内容：若分镜有 dialogue 字段，检查 prompt 中是否含 `说:"` 或 `说：`

```

```

--- 参数 ---
时长：{duration}s | 宽高比：{aspect_ratio}
```

### 注意事项

- 台词是提示词的一部分，必须嵌入在 prompt 正文中（格式：`角色+动作+说:"台词"`），而非仅存在于独立的 dialogue 字段
- references 中的 @引用标记是即梦使用时上传参考图的对应说明，需完整保留
- 复制时保留换行格式，确保粘贴后可读性

## 十七、音乐线索（Music Cue）生成规范

### 概述

每个分镜可包含 `music_cue` 字段，标注该段视频的背景音乐风格建议，供后期合成时选曲参考。

### music_cue 字段格式

```json
{
  "music_cue": {
    "mood": "紧张/悲伤/激昂/温馨/神秘/史诗/日常",
    "intensity": "低/中/高",
    "instruments": "古筝/二胡/大鼓/弦乐/管弦/无",
    "tempo": "缓慢/中速/快速",
    "transition": "渐入/渐出/突入/突止/持续"
  }
}
```

### 场景-音乐映射表

| 场景情绪 | mood | intensity | instruments | tempo |

### 音乐线索生成规则

1. **情绪匹配**：music_cue.mood 应与分镜的情绪标签一致
2. **强度递进**：同一场戏内 intensity 应有变化（低→中→高），避免平铺
3. **转场衔接**：`fade_black` 转场前的分镜 music_cue.transition 设为 `渐出`
4. **高潮标记**：情感爆发点的分镜 intensity 设为 `高`
5. **静默对比**：在高潮前设置一个 `无音乐` 的分镜，利用静默增强冲击

### 在视频提示词中的表达

音乐线索**不写入 video_prompt**（Seedance 2.0 的音乐由 @音频引用 控制），但应在分镜脚本中标注：

```
--- 音频提示 ---
背景音乐：[mood] [tempo]，[instruments]
音效：[具体环境音/动作音效]
```
